26/**
27 * Segregates zero-copy/SHM implementations of concepts residing in parent namespace ipc::transport::struc proper --
28 * most notably the concepts ipc::transport::struc::Struct_builder and ipc::transport::struc::Struct_reader --
29 * and items related to them. This namespace proper is focused on struc::shm::Builder and struc::shm::Reader,
30 * templates which enable (in a SHM-provider-parameterized way) zero-copyable structured messages by implementing
31 * the aforementioned pair of concepts respectively.
32 *
33 * Sub-namespaces, including struc::shm::classic and struc::shm::arena_lend::jemalloc as of this writing,
34 * pertain to SHM-provider-specific items relevant (at least) to that pair of templates (`Builder` and `Reader`).
35 * In particular, e.g., struc::shm::classic::Builder is the concrete type that implements
36 * a zero-copy `Struct_builder` using the ipc::shm::classic SHM-provider (and similarly for `Reader`). For user
37 * convenience and generic coding ease, each SHM-provider shall provide that pair of aliases (`Builder` and `Reader`)
38 * in a sub-namespace in some reasonable way mirroring the SHM-provider core's namespace path. (Hence note that
39 * SHM-classic provider core is in ipc::shm::classic, hence the `Builder` and `Reader` aliases are in
40 * ipc::transport::struc::shm::classic accordingly.)
41 *
42 * (If that didn't read like some sort of comprehensible English, you probably need not worry about it. In particular
43 * if you use ipc::session to set up your IPC, you'll have access to higher-level aliases to make it all simple --
44 * while internally they'll most likely use the stuff in the preceding paragraph. E.g.,
45 * ipc::session::shm::classic::Session_mv::Structured_channel is your guy... no `Builder`s or `Reader`s in sight
46 * about which you would need to worry.)
47 */

67/**
68 * Utility that saves the result of a `Shm_session1::lend_object<T>(const shared_ptr<T>&)` result into
69 * the given capnp-generated `ShmHandle`-typed field. On the deserializing end, one
70 * can get back this value via capnp_get_shm_handle_to_borrow() and pass it to
71 * `Shm_session2::borrow_object<T>()` to yield a `shared_ptr<T>` equivalent to the original passed to `lend_object()`.
72 *
73 * `Shm_session1` and `Shm_session2` supported, respectively, include:
74 * - shm::classic::Pool_arena, shm::classic::Pool_arena (sic);
75 * - shm::arena_lend::jemalloc::Shm_session, shm::arena_lend::jemalloc::Shm_session (sic);
76 * - session::shm::classic::Client_session, session::shm::classic::Server_session;
77 * - vice versa;
78 * - session::shm::arena_lend::jemalloc::Client_session, session::shm::arena_lend::jemalloc::Server_session;
79 * - vice versa;
80 * - shm::Builder::Session pointee, shm::Reader::Session pointee.
81 *
82 * @param shm_handle_root
83 * Non-null (or behavior undefined/assertion may trip) pointer to `ShmHandle` builder to mutate.
84 * @param lend_result
85 * What `lend_object<T>()` returned. Not `.empty()`, or behavior undefined (assertion may trip).
86 */
87void capnp_set_lent_shm_handle(schema::ShmHandle::Builder* shm_handle_root,
88 const flow::util::Blob_sans_log_context& lend_result);
89
90/**
91 * Utility that's the reverse of capnp_set_lent_shm_handle() to be invoked on the deserializing side.
92 *
93 * @param shm_handle_root
94 * `ShmHandle` reader to access. Behavior undefined if was not set by capnp_set_lent_shm_handle().
95 * @param arg_to_borrow
96 * Shall be set to what to pass to `borrow_object<T>()` (non-null pointer or behavior undefined/assertion
97 * may trip). `arg_to_borrow->get_logger()` shall not be modified; so set it what you want, if you want.
98 */
99void capnp_get_shm_handle_to_borrow(const schema::ShmHandle::Reader& shm_handle_root,
100 flow::util::Blob_sans_log_context* arg_to_borrow);
