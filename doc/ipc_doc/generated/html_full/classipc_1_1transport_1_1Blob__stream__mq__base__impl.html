If Blob_stream_mq_sender_impl sends an empty message, in NORMAL state Blob_stream_mq_receiver enters CONTROL state and expects one of these values in the next message, to react as documented per enum value (upon re-entering NORMAL state).

Enumerator
S_END_SENDING 

Indicates sender user invoked Blob_sender::end_sending() (graceful close). Emit/queue graceful-close to user.

S_PING 

Indicates sender user invoked auto_ping() earlier, and this is an incoming periodic ping.

If receiver user invoked idle_timer_run() earlier, reset countdown to death due to lacking in-pings.

S_END_SENTINEL 

Sentinel: not a valid value. May be used to, e.g., ensure validity of incoming value of allegedly this type.

Internal helper for Blob_stream_mq_sender and Blob_stream_mq_receiver that operates both the start and end of the anti-dupe-endpoint machinery used by those 2 classes to prevent more than 1 _sender and more than 1 _receiver for a given underlying MQ.

The input is a handle to the MQ, as created by user as of this writing, either on the sender or receiver end (snd_else_rcv). The output is one of:

  • a null return, indicating that either there's already a sender (or receiver) already registered (typically), or some other (unlikely) system error. *err_code will indicate what happened; or
  • a non-null returned unique_ptr that has moved mq into itself/taken over ownership of it, with certain machinery such that when that unique_ptr (or moved version thereof) is .reset() or destroyed, the proper deinit will run, making it possible to create another MQ at the same Shared_name.

mq is untouched if null is returned.

Parameters
logger_ptrLogger to use subsequently. Errors are logged as WARNING; otherwise nothing of INFO or higher verbosity is logged.
mqSee above.

err_codeMust not be null, or behavior undefined; will be set to success if non-null returned or reason for failure otherwise.
Returns
null or pointer to new Persistent_mq_handle moved-from mq, and with deinit deleter.
